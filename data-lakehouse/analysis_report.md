## 5. 用户行为漏斗

```
浏览用户 (有 pv)
    │
    ├── 仅浏览未做任何其他行为 ─── 21 用户 (0.2%)
    │
    ├── 收藏 (有 fav) ─────────── 57 用户 (0.6%)
    │
    ├── 加购 (有 cart) ────────── 916 用户 (9.2%)
    │       │
    │       └── 其中未购买 ────── 约 916 - (26,011含购买用户) = 高漏斗流失
    │
    └── 购买 (有 buy) ─────────── 9,006 用户 (90.0%)
```

**关键发现**：90% 的活跃用户产生了购买行为！这是数据集本身的特性（脱敏后仅保留有购买记录的用户行为）。

## 7. 技术架构说明

### 数据管道
```
CSV (UserBehavior.csv)
    ↓ psycopg2 COPY
PostgreSQL (staging_user_behavior)
    ↓ pyiceberg REST API
Apache Iceberg (MinIO s3a://warehouse/)
    ↓ Trino SQL 查询
分析报表
```

### 关键组件
- **Iceberg REST**: `http://iceberg-rest:8181`（管理表元数据）
- **MinIO**: `localhost:9000`（存储 Parquet 文件）
- **Trino 459**: `localhost:8080`（Iceberg catalog 查询引擎）
- **PostgreSQL**: `localhost:5432`（staging 数据，ICEberg JDBC catalog）

### 已知问题
- Flink Kafka source 无法从 Flink TM 消费（Kafka advertised listener 网络问题）
- 已绕过：数据直接从 CSV → PostgreSQL → Iceberg，不依赖 Flink Streaming
