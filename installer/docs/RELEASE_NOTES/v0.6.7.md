## v0.6.7 - Agent rewrite bug fixes + v0.6.6 carryover

xhs-saas-console v0.6.7 - agent_rewrite fixes (persona + analysis fallback + fan-out) + all v0.6.6 features (SiliconFlow, ai_mode, scheduler fix)

### Highlights

- **`agent_rewrite` persona propagation** (bug fix): the `persona` parameter
  of `factory.agent_rewrite()` is now actually injected into the prompt that
  reaches the Coordinator/ContentAgent. Previously the function accepted
  `persona` but silently dropped it before building the task string, so the
  user-configured `task.ai_persona` never reached the LLM. (`commit: 1a02902`)
- **`agent_rewrite` analysis preservation** (bug fix): when the Coordinator
  returns an analysis-only result without a draft (e.g. multi-intent fan-out
  where the user asked only for analysis), the analysis is now preserved in
  `content.extra["agent_analysis"]` instead of being silently discarded.
  The audit log now sees what the agent produced.
- **Coordinator fan-out partial-failure tolerance** (bug fix): single
  failing sub-graph no longer aborts the entire fan-out. Switched from
  `asyncio.gather(*tasks)` to `asyncio.gather(*tasks, return_exceptions=True)`
  in `CoordinatorAgent.fan_out`. Surviving sub-graphs still contribute;
  the synthesise node naturally skips exception entries. (`commit: 1a02902`)
- **14 new tests** in three files:
  - `tests/test_factory_agent_rewrite.py` (6 cases)
  - `tests/test_coordinator_partial_failure.py` (3 cases)
  - `tests/test_scheduler_ai_mode.py` (5 cases)
- **TYPE_CHECKING for Task in factory.py** (ruff F821 compliance).
- **Carryover from v0.6.6 (not previously released):**
  - SiliconFlow support: `OPENAI_BASE_URL` env var now read via
    `AISettings.validation_alias` and forwarded to `ChatOpenAI` /
    `OpenAIEmbeddings`. Any OpenAI-compatible proxy works (SiliconFlow,
    Groq, LM Studio, ollama).
  - `Task.ai_mode` field: new `tasks.ai_mode` column (`rewrite` | `agent`).
    `rewrite` = single-prompt `factory.maybe_rewrite()` (fast, default).
    `agent` = `CoordinatorAgent.coordinate_task()` fan-out.
  - `factory.agent_rewrite()`: new async factory that calls the
    CoordinatorAgent for structured multi-agent content generation.
  - Scheduler fix: `SchedulePostTool` no longer crashes at runtime because
    of a missing `get_scheduler` symbol. Added `get_scheduler()` to
    `app/scheduler/__init__.py`.
  - `tests/test_real_llm.py`: 6 new smoke tests for SiliconFlow-style
    setups — auto-skip when no `OPENAI_API_KEY`.
