## v0.6.4 - XHS publish flow + console polish

xhs-saas-console v0.6.4 - XHS publish flow + console polish

### Highlights

- XHS publish flow now routes through `/new/home` and clicks `发布图文笔记` before navigating to the publish form, matching the actual creator portal SPA flow (commit dde93bc).
- `scripts/rate_limit_probe.py` refactored: `OUT_DIR` resolved from repo root, correct middle-dot task name, async `load_account_and_task`, and `_Acct` stand-in for adapter.publish calls (commit b933e14).
- DOM inspector (`xiaohongshu-saas/scripts/_inspect_publish_dom.py`) confirms the publish page only exposes one editable element after click (`<input type="file" class="upload-input">`); used to diagnose the v0.6.3 "publish form wait timeout" failures.

### Quick start (MSI)

```
msiexec /i xhs-saas-console-0.6.4.msi /qn
msiexec /i xhs-saas-console-0.6.4.msi /qn INSTALLDIR=%USERPROFILE%\Apps\xhs-saas-console
msiexec /x {CB49ED5C-30E2-4F98-99DC-63A15816DC5E}
```

### Known issues / warnings (carryover)

- DOM probe shows the publish page only renders the file input - the title / body / topic editors load asynchronously after the upload fires. The publish adapter still needs to be updated to wait on `input.upload-input` instead of the previous combined selector.
- `scripts/rate_limit_probe.py` still hangs under Chromium when run from the `xiaohongshu-saas/` cwd (AMSI / OOM observed in Cursor Agent PS host - unrelated to this release).
