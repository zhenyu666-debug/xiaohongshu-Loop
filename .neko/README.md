# Neko shared room

Stand up a self-hosted virtual browser the friend can open in their browser and drive,
so we don't have to deal with ngrok (AlibabaProtect blocks it, NM-7) or cloudflared
(GFW blocks GitHub downloads).

Layout:
- `docker-compose.yml` — pulls `ghcr.io/m1k1o/neko/chromium:latest`, exposes :8080.
- `passwords/admin_password` + `passwords/user_password` — generated by `setup-secrets.ps1`.
- `setup-secrets.ps1` — re-rolls the two passwords (admin / friend), updates `passwords/`.
- `share-neko-supervisor.ps1` — one-shot bring-up + watch loop. Brings everything up
  (neko + backend uvicorn + vite + serveo SSH tunnel), captures the share URL into
  `current_url.txt`, then re-checks every 10s and restarts anything that died. Ctrl-C
  to stop; the children stay alive.
- `healthcheck.ps1` — emits a JSON snapshot of the stack (`neko` / `backend` /
  `frontend` / `serveo.tunnels` / `serveo.share_url`) to stdout AND to
  `supervisor_state.json`. Exits 1 if anything is down so cron / Uptime Kuma can
  alert on it.
- `logs/` — supervisor + child process logs (created at runtime).

## Bring up (recommended — use the supervisor)
```
powershell -NoProfile -ExecutionPolicy Bypass -File .neko/share-neko-supervisor.ps1
```
Tail logs: `Get-Content .neko/logs/supervisor.log -Wait`.

## Bring up (manual — without supervisor)
```
docker compose -f .neko/docker-compose.yml --env-file .neko/.env up -d
powershell -NoProfile -ExecutionPolicy Bypass -File .neko/share-neko.ps1
```

## Bring down
```
docker compose -f .neko/docker-compose.yml down
Get-Process -Name ssh,python,node -ErrorAction SilentlyContinue | Where-Object { $_.CommandLine -match 'uvicorn|vite.cmd|ssh.*serveo' } | Stop-Process -Force
```

## Health check
```
powershell -NoProfile -ExecutionPolicy Bypass -File .neko/healthcheck.ps1
```
Returns JSON; exits 1 on degraded. Wire into your scheduler / monitor of choice.

## Get URLs/credentials
```
cat .neko/current_url.txt             # serveo share URL — friend opens this
cat .neko/passwords/admin_password    # owner drive login
cat .neko/passwords/user_password     # friend watch-only login
```

Notes:
- The neko container calls `host.docker.internal:5173`. On Windows + Docker Desktop that
  resolves to the WSL2 virtual gateway, which routes to the host's loopback / nic.
  Vite must be listening on `0.0.0.0:5173` (default in vite 6+) AND must allow the
  `host.docker.internal` Host header. `frontend/vite.config.ts` sets
  `server.allowedHosts: true` to satisfy this; without it chromium's first GET to
  the autostart URL returns 403 with 0 bytes.
- The Chromium room autostart opens the autostart URL with the Chromium user logged in.
  In the GUI, log in to the room with admin creds to drive; friend joins with user creds.
- `TG_HOST=127.0.0.1 TG_RESTPP_PORT=19999` is set by the supervisor so the backend's
  `/api/health` ping to TigerGraph RESTPP fails fast (WinError 10061 in ~1s) instead of
  blocking 14s waiting for an unreachable TigerGraph. The full `/api/health` response
  still returns 200 with `tigergraph.status="unreachable"` — frontend behaviour is
  unchanged.
